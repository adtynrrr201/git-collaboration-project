# Examples

This directory contains example README files generated by the Awesome README Generator.

## Available Examples

### 1. Minimal Template
- **File**: `minimal-example.md`
- **Use case**: Simple projects with basic documentation needs
- **Features**: Clean, straightforward structure

### 2. Modern Template
- **File**: `modern-example.md`
- **Use case**: Professional projects with badges and structured sections
- **Features**: Badges, API documentation, contributing guidelines

### 3. Detailed Template
- **File**: `detailed-example.md`
- **Use case**: Complex projects requiring comprehensive documentation
- **Features**: Table of contents, extensive API reference, examples

## How to Generate

```bash
# Generate with minimal template
awesome-readme generate --template minimal

# Generate with modern template (default)
awesome-readme generate --template modern

# Generate with detailed template
awesome-readme generate --template detailed
```

## Customization

After generating a README, you can customize it by:

1. Adding project-specific sections
2. Including screenshots or diagrams
3. Updating badges and links
4. Adding installation instructions for specific platforms
5. Including troubleshooting information

## Tips

- Keep your README concise but comprehensive
- Use clear headings and structure
- Include examples for common use cases
- Add badges for build status, coverage, etc.
- Keep documentation up to date with code changes
